# Moderation Rubric

This document exists because Topey intentionally allows flexible moderator judgment. Without a written spine, moderation quality drifts fast.

## Public Listing Standard

A place should only become public when a moderator believes all of the following are true:

1. The location is specific enough that a user can find the same place again.
2. The listing describes a repeatable place with stable rules, not a one-off hiding spot.
3. Allowed actions are explicit.
4. Restrictions are explicit.
5. There is a named proof basis and a written note explaining why the place is trusted.

## Evidence Types

- `Moderator scout note`
- `Founder scout note`
- `Operator confirmation`
- `Trusted user observation`

## Approval Checklist

- Pin is precise enough
- Neighborhood matches the pin and description
- At least one allowed action is stated clearly
- At least one restriction is stated clearly
- Best time is specific
- Evidence note explains why the moderator trusts the listing
- The listing does not read like a random loophole or evasive hiding spot

## When To Use `needs_more_proof`

Use `needs_more_proof` when:

- the pin is too vague
- the proof note is weak or stale
- rules are incomplete
- the place might be real but the listing is not yet defensible

## When To Reject

Reject when:

- the submission is fundamentally a random hiding spot
- no stable permission or tolerance pattern is described
- the listing cannot be defended as a repeatable place
- the pin or rules are so weak that the queue would just churn forever

## Live Edit Standard

Trusted-user live edits are allowed, but moderators should review the audit log regularly for:

- abrupt rule changes
- confidence drift without fresh proof
- edits that remove restrictions
- edits that push a place toward vagueness

If a trusted scout becomes sloppy, revoke the trusted role before the map quality drifts.
